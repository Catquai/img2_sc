# 国旗参考图微调规则

- 图片-only、附件-only、路径-only 的国旗参考图默认直接按 `reference_only` 生成轻微变体；除非用户明确说只分析或不生成，不要追问。
- 文件夹路径默认按 `batch_reference_folder` 处理：自动识别其中 `.png`、`.jpg`、`.jpeg`、`.webp`、`.bmp` 参考图并逐张处理；默认不递归，除非用户明确要求包含子文件夹。
- 批量处理时每张图必须独立分析尺寸、alpha、圆角、外形和国旗元素，并独立产出 `final_json`；不要把第一张图的参数套用到整批图片。
- 批量输出不得覆盖源文件。若用户指定输出目录则使用该目录；否则创建独立输出目录，建议命名为 `<原文件夹名>_flag_output`。
- 批量输出必须按参考图原名称保存：保留原始 basename 和扩展名；如必须转换为透明 PNG，则保留 basename 并改用 `.png`。
- 输出目录存在同名文件时不要静默覆盖；除非用户明确允许覆盖，否则使用独立批次目录或追加安全后缀，并说明命名策略。
- 必须以参考图为准，不主动查找、纠正或替换为标准国旗版本。
- 生成或确定性重建前必须产出 `final_json`，记录画布、外形、alpha、国旗元素、调色策略、不变约束和失败标准。
- 必须保持最终尺寸与参考图完全一致。
- 必须检查并复用参考图的透明度：透明背景、半透明抗锯齿边缘、圆角外透明区、内部半透明层。
- 必须检查圆角大小。圆角矩形要记录半径像素或相对比例；圆形图标要保持圆形 alpha 遮罩。
- 必须保持国旗布局、元素数量、条纹顺序、徽章、星星、文字、层级、比例、位置和裁切关系。
- 只允许轻微随机调色：小幅色相、小幅饱和度、小幅明暗变化。
- 不允许让明暗、对比度、饱和度或色彩身份与参考图出现明显差距。
- 不允许添加参考图没有的纹理、阴影、渐变、3D、高光、描边、边框、贴纸效果或新装饰。
- 简单几何国旗优先用确定性绘制或像素级后处理；图像生成只作为必要时的辅助手段。
- 结构错误不能靠裁切、拉伸、挤压或强套 alpha mask 修复；国旗元素、裁切或外形错了就重新生成或重建。
