# img2-sc-frame

## 核心规则

只对现有帧动画图片资源改色。必须保持帧数、帧顺序、画布尺寸、单帧块尺寸、alpha 透明通道、透明留白、圆角遮罩、元素位置、层级关系、运动轨迹，以及所有非颜色类的逐帧差异。

不要重新设计图标，不要改变动画姿态，不要新增高光、纹理、阴影、模糊、装饰、文字或 3D 效果，不要裁切内容，也不要把帧动画改成看起来像静态资源的形态。

## 输入类型判定

处理前先判定输入类型：

- `frame_sequence`：多张同尺寸图片，表示连续动画帧。
- `sprite_strip_horizontal`：横向精灵图条，单张图片横向排列多个等尺寸帧，从左到右读取。
- `sprite_strip_vertical`：纵向精灵图条，单张图片纵向排列多个等尺寸帧，从上到下读取。
- `grid_sprite`：网格精灵图，单张图片按行列排列多个等尺寸帧，按资源原有顺序读取。
- `single_frame`：单张图片但用户明确要求按动画资源规则改色，或该图属于动画资源的一帧。
- `batch_reference_folder`：用户提供文件夹路径，或明确要求批量处理文件夹内参考图。必须自动识别文件夹内可处理图片资源，并按文件夹批量处理规则执行。

如果用户提供多张尺寸一致、主体一致、只在高光位置、旋转角度、透明度、偏移、扫光、进度状态等细节上有差异的图片，默认把它们判定为同一组帧动画序列，除非用户明确说明它们是互不相关的备选图。

## 默认意图

当用户只提供图片、图片附件、图片路径或一组帧动画资源，而没有提供目标颜色、改色方向或其他文本描述时，不要追问。直接执行随机改色：

- 随机选择一个与源主题色有明显区分的新主题色系。
- 对同一组帧动画资源应用同一套全局随机色彩映射，避免逐帧闪烁。
- 保持源图明度层级、渐变、高光、阴影、发光、透明度和抗锯齿关系。
- 默认只随机改主要主题色或高饱和色系，中性色、透明背景、白色符号、黑灰阴影和文字保持不变。
- 如果随机到青色、绿色、橙黄色、黄色等高感知明度色系，必须主动压低目标色明度，不要生成过亮的荧光色或浅色底。
- 随机色不能过暗、过亮、过灰或与源色过于接近；如果一眼看不出改色差异，视为失败。

当用户只提供文件夹路径，或说“处理这个文件夹 / 批量处理 / 文件夹内图片”时，不要追问。直接执行 `batch_reference_folder`：自动识别文件夹内图片资源，按批量规则分组并改色。

## 文件夹批量处理

- 自动识别文件夹内常见图片格式：`.png`、`.jpg`、`.jpeg`、`.webp`、`.bmp`、`.gif`。默认跳过非图片文件、隐藏文件、临时文件和无法读取的损坏图片。
- 默认只处理用户提供文件夹的直接子文件；只有用户明确要求“递归 / 包含子文件夹”时，才递归处理子目录。
- 文件夹内多张同尺寸、文件名带连续编号、主体一致且仅动画状态不同的图片，默认识别为同一组 `frame_sequence`，对整组应用同一套全局颜色映射。
- 文件夹内多个精灵图条、网格精灵图或尺寸/主体明显不同的图片，默认视为多个独立资源逐项批量处理。每个资源必须独立分析帧布局、alpha、扫光和改色映射。
- 同一批次中不要把第一张图的尺寸、帧数、精灵图布局、alpha 或颜色映射无条件套用到其他独立资源；只有被判定为同一组 `frame_sequence` 的文件才共享全局映射。
- 批量输出不得覆盖源文件。若用户指定输出目录，则输出到该目录；否则在参考图文件夹旁创建独立输出目录，建议命名为 `<原文件夹名>_frame_output`。
- 输出文件必须按参考图原名称保存：保留原始 basename 和扩展名，例如 `loading_01.png` 输出仍命名为 `loading_01.png`。如必须转换格式以保留透明 PNG，则保留 basename，扩展名改为 `.png`。
- 对被识别为同一组独立帧序列的文件，输出数量和文件名集合必须与源序列一致，帧顺序必须由原文件名排序保持。
- 如果输出目录中已存在同名文件，不要静默覆盖；除非用户明确允许覆盖，否则使用独立批次目录或追加安全后缀，并在结果说明中列出命名策略。
- 批量处理完成后必须汇总成功、失败和跳过的文件列表。失败文件不应阻塞整批任务，除非全部失败或工具不可用。

## 工作流程

1. 检查源资源或文件夹几何信息。
   - 单图输入按单个资源处理。
   - 文件夹输入先枚举可处理图片，判断它们是同一组 `frame_sequence` 还是多个独立资源，再建立批量任务列表。
   - 记录整体画布宽高。
   - 对精灵图条或网格精灵图，记录单帧宽高、行数、列数、帧数、帧间距和透明留白。
   - 对独立帧序列，记录每帧路径、尺寸、索引和文件名顺序。
   - 检查 alpha 透明通道：全透明像素、半透明抗锯齿边缘、柔和阴影、发光、圆角外部透明遮罩。

2. 建立帧地图。
   - 识别所有帧共享的静态底图和固定元素。
   - 识别动画元素：扫光、旋转弧、移动圆点、发光脉冲、透明度脉冲、加载条、流光扫动、逐帧角标等。
   - 对按钮扫光资源，必须把主体底色和扫光带分成两个逻辑层记录。扫光带通常表现为斜向或横向的浅色半透明区域，随帧移动或改变位置。
   - 记录哪些像素或元素允许随帧变化，哪些元素必须锁定。
   - 记录改色映射：源色相/色域 -> 目标色相/色域，并保持相对明度、饱和度层级和透明度关系。

3. 使用破坏性最低的方法改色。
   - 当原图形状、构图和动画正确时，优先使用确定性的色相、饱和度、通道或色域映射。
   - 默认精确保留 alpha 透明通道，不要修改透明度，除非用户明确要求改变透明度。
   - 保留局部明度、渐变层次、阴影透明度、模糊半径、高光关系和抗锯齿边缘。
   - 单一主题色 UI 资源只映射该主题色系，透明背景、白色字形、黑灰阴影、禁用态灰色等中性色默认锁定。
   - 多色资源只改用户指定色系，或按目标色板一一映射；不要把多种源色合并成单一纯色，除非用户明确要求。
   - 对青色、绿色、橙黄色、黄色等人眼感知偏亮的目标色，不要只保持 HSL 明度；必须按感知明度压暗，让白色文字或白色图形放在按钮上时仍清晰。
   - 对扫光必须单独调色和调明度：主体底色负责整体颜色识别，扫光负责高光质感。不要把扫光和主体用同一个明度压缩结果处理。

## 扫光分层规则

按钮类帧动画中出现扫光、流光、斜向亮带或移动高光时，必须把扫光作为独立视觉层处理：

- 扫光位置、宽度、倾斜角度、羽化边缘、透明度变化和逐帧移动轨迹必须与源图一致。
- 扫光必须比主体底色明显，但不能接近纯白。建议扫光平均感知明度比相邻主体区域高 `0.10-0.22`，峰值可高一些，但不能导致白色前景局部消失。
- 扫光颜色必须与目标主题色同系或邻近色系，通常使用同色相的浅亮版本，或轻微偏暖/偏冷的高光色；不要混入互补色导致发灰、发脏。
- 扫光 HSL 饱和度是适用于所有目标色相的通用参数，不得按蓝色、绿色、红色等单一色相写死处理范围。常规主题默认将扫光 HSL `S` 控制在 `60%-75%`。主体本身为低饱和、粉彩或品牌灰调时允许适当下调，但必须保留可辨识的目标色相身份，且不得出现灰白、灰雾、米灰、泥色或脏色观感。
- 调整扫光饱和度时，优先只修改 HSL `S`，保持扫光的 HSL `H` 与目标主题同系，并锁定原有 HSL `L`、alpha、羽化边缘和逐帧轨迹；如果提高饱和度后感知亮度明显变化，再做最小幅度的 `L` 补偿。
- 青色、绿色、橙黄色、黄色主体需要降明度时，扫光可以独立保留更高明度，但必须保持干净明亮；不要因为主体压暗而让扫光变得不明显。
- 粉色、红色、紫色主体的扫光也要保持清晰色相，不要变成灰粉、灰紫或低饱和雾带。
- 如果扫光在预览里看不出来、与主体差异太小、边界糊成脏色、或让整体按钮显得灰蒙蒙，判定失败。
- 如果扫光过亮导致未来白色文字或白色图形经过该区域时明显丢失边缘，判定失败。

4. 仅在必要时使用图像生成。
   - 只有确定性改色无法满足材质、风格或复杂渐变变化时，才使用图像生成辅助。
   - 生成提示词必须强调：帧数、帧顺序、单帧尺寸、整体画布、alpha 透明通道、运动轨迹、逐帧姿态和所有非颜色差异必须与参考图一致。
   - 生成后尽量恢复源图的精确尺寸和 alpha 遮罩。
   - 如果生成结果新增形状、改变动画姿态、破坏精灵图布局或改变透明边缘，必须判定失败并重做。

5. 交付前验证。
   - 最终整体图片尺寸必须与源图逐像素一致。
   - 精灵图条的帧数、单帧尺寸、行列布局、间距和读取顺序必须与源图一致。
   - 独立帧序列的输出数量和每帧尺寸必须与源序列一致。
   - alpha 透明通道必须与源图一致，除非用户明确要求透明度变化。
   - 非目标色区域必须保持视觉不变。
   - 如果资源可能承载白色文字或白色图形，目标色背景必须保留足够对比度；青色、绿色、橙黄色、黄色改色后若过亮导致白色前景识别性下降，判定失败。
   - 扫光必须独立检查：它要比主体明显、色彩干净、边缘自然，且不能出现发灰、发脏、变泥、过暗、不明显或过曝的问题。
   - 按顺序预览时，动画运动必须仍然读作同一个动画。

## 改色约束

允许：

- 改变用户指定源色范围的色相。
- 轻微调整饱和度以匹配目标色。
- 保持局部明度差异，让渐变、倒角、抗锯齿和高光仍然自然。
- 保持源 alpha 透明通道、阴影强度和发光强度。
- 对所有帧应用同一套全局颜色映射，避免闪烁。
- 对青色、绿色、橙黄色、黄色做适度降明度处理，优先保证白色字体、白色图标、白色符号叠加时的识别性。
- 对扫光层单独设置明度、饱和度和色相，确保扫光明显且干净，不被主体色吞掉。

禁止：

- 改变画布尺寸、单帧尺寸、帧数、帧顺序或精灵图布局。
- 移动、旋转、缩放、重绘、简化或替换任何形状。
- 未经要求改变 alpha 遮罩、圆角、透明留白、抗锯齿边缘、阴影透明度或发光范围。
- 把渐变、发光、阴影、高光改成扁平纯色。
- 新增纹理、噪点、3D 效果、描边、徽章、文字、装饰或额外光照。
- 未经要求改动白色、黑色、灰色、透明背景、阴影、禁用态等中性层。
- 对每帧分别自由调色，导致目标色在帧间不一致或闪烁。
- 把青色、绿色、橙黄色、黄色改得过亮，导致白色文字、白色图形或白色图标叠加后看不清。
- 把扫光改得不明显、发灰、发脏、发泥、低饱和，或让扫光与主体底色融成一片。
- 把扫光改得过亮或过白，导致叠加白色前景时局部识别性下降。
