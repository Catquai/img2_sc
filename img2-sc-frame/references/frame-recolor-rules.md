# 帧动画改色规则

- 多张同尺寸连续 PNG 默认视为同一组动画帧，除非用户说明它们互不相关。
- 用户只提供图片、图片附件、图片路径或文件夹路径且没有文本描述时，不要追问目标色；直接随机改主要主题色。
- 文件夹路径默认按 `batch_reference_folder` 处理：自动识别其中 `.png`、`.jpg`、`.jpeg`、`.webp`、`.bmp`、`.gif` 图片资源；默认不递归，除非用户明确要求包含子文件夹。
- 文件夹内多张同尺寸、文件名带连续编号、主体一致且仅动画状态不同的图片，默认识别为同一组 `frame_sequence`，整组应用同一套全局颜色映射。
- 文件夹内多个精灵图条、网格精灵图或尺寸/主体明显不同的图片，默认视为多个独立资源逐项批量处理；每个资源必须独立分析帧布局、alpha、扫光和改色映射。
- 批量输出不得覆盖源文件。若用户指定输出目录则使用该目录；否则创建独立输出目录，建议命名为 `<原文件夹名>_frame_output`。
- 批量输出必须按参考图原名称保存：保留原始 basename 和扩展名；如必须转换为透明 PNG，则保留 basename 并改用 `.png`。
- 同一组独立帧序列的输出数量和文件名集合必须与源序列一致，帧顺序必须由原文件名排序保持。
- 输出目录存在同名文件时不要静默覆盖；除非用户明确允许覆盖，否则使用独立批次目录或追加安全后缀，并说明命名策略。
- 横向、纵向或网格精灵图必须按固定帧布局处理；不要改变单帧块数量、单帧块尺寸、间距或读取顺序。
- 默认精确保留源 alpha 透明通道，包括全透明像素、半透明抗锯齿、圆角遮罩、阴影和发光。
- 所有帧必须应用同一套全局颜色映射，避免逐帧色差造成闪烁。
- 保持明度和透明度关系，让渐变、倒角、边缘平滑和高光仍然像原生资源。
- 青色、绿色、橙黄色、黄色属于容易变得过亮的色系，必须适当降低感知明度，避免白色字体或白色图形叠加后识别性下降。
- 扫光必须作为独立层处理：位置、宽度、倾斜角度、羽化和逐帧轨迹不变；明度要比主体明显，颜色要干净、同系偏亮，不得发灰、发脏、发泥或被主体色吞掉。
- 默认锁定中性层：透明背景、白色前景符号、黑灰阴影、禁用态灰色和文字。
- 只改用户指定的色相范围或色板槽位。
- 随机改色时必须选择与源主题色明显不同的新色系，并对全部帧使用同一映射。
- 不要重绘形状、调整运动姿态、添加细节、简化动画、裁切内容或改变画布尺寸。
- 交付前必须验证输出尺寸、帧数、帧顺序、逐帧 alpha 透明通道和动画连续性。

出现以下情况即判定失败：

- 输出尺寸与源图不同。
- 帧数或精灵图布局与源图不同。
- alpha 透明边缘、透明留白、圆角、阴影透明度或发光范围意外变化。
- 动画路径、姿态、旋转、透明度脉冲、扫光位置或加载状态变化。
- 目标色在帧间不一致，导致播放时闪烁。
- 青色、绿色、橙黄色、黄色过亮，导致白色文字、白色图形或白色图标看不清。
- 扫光不够明显、颜色变脏、变灰、变泥、低饱和，或与主体底色融成一片。
- 扫光过亮或过白，导致白色前景经过扫光区域时局部看不清。
- 新增装饰、纹理、描边、文字、图标身份或额外光照。
